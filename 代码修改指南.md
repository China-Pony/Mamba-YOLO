### A5.2 Neck XSSBlock 中的 3×3 stride=2 下采样卷积

**YAML 中的位置**（[Mamba-YOLO-B.yaml L30](file:///d:/Desktop/Codes/v1/Mamba-YOLO/ultralytics/cfg/models/mamba-yolo/Mamba-YOLO-B.yaml#L30)、[L34](file:///d:/Desktop/Codes/v1/Mamba-YOLO/ultralytics/cfg/models/mamba-yolo/Mamba-YOLO-B.yaml#L34)）：

```yaml

  - [-1, 1, Conv, [512, 3, 2]]
```

**处理方式**：FDConv 默认 `use_fdconv_if_stride_in=[1]`（[L526](file:///d:/Desktop/Codes/v1/FDConv/FDConv_detection/mmdet_custom/FDConv.py#L526)），stride=2 卷积会自动 fallback 到普通 Conv。所以这里**不需要特殊处理**——直接替换为 FDConv 即可，框架会自动判断。

但更推荐的做法是：为 stride=2 的 Conv 设置 `use_fbm_for_stride=True`（[L528](file:///d:/Desktop/Codes/v1/FDConv/FDConv_detection/mmdet_custom/FDConv.py#L528)），让 FBM 在 stride=2 时也生效：

```python

FDConv(256, 256, 3, stride=2, padding=1,
       use_fbm_for_stride=True,
       kernel_num=4)
```

实际实现时，需要在 `parse_model` 中处理 `FDConv` 的实例化，或者创建一个 `ConvFD` 包装类。

### A5.3 创建 `ConvFD` 包装类

**文件**：[`fdconv.py`](file:///d:/Desktop/Codes/v1/Mamba-YOLO/ultralytics/nn/modules/fdconv.py) 末尾添加：

```python
class ConvFD(nn.Module):
    """Conv-BN-Act with FDConv replacing standard Conv2d."""
    default_act = nn.SiLU()

    def __init__(self, c1, c2, k=1, s=1, p=None, g=1, d=1, act=True,
                 kernel_num=4, param_ratio=1, use_fbm=True):
        super().__init__()
        self.conv = FDConv(
            c1, c2, k, s,
            padding=k // 2 if p is None else p,
            groups=g, dilation=d, bias=False,
            kernel_num=kernel_num,
            param_ratio=param_ratio,
            use_fdconv_if_c_gt=16,
            use_fdconv_if_k_in=[1, 3],
            use_fdconv_if_stride_in=[1, 2],
            use_fbm_if_k_in=[3] if use_fbm else [],
            use_ksm_local=True,
        )
        self.bn = nn.BatchNorm2d(c2)
        self.act = Conv.default_act if act is True else \
                   act if isinstance(act, nn.Module) else nn.Identity()

    def forward(self, x):
        return self.act(self.bn(self.conv(x)))

    def forward_fuse(self, x):
        return self.act(self.conv(x))
```

这样在 YAML 中可以直接将 `Conv` 替换为 `ConvFD`：

```yaml
  - [-1, 1, ConvFD, [256, 3, 2]]    # 使用 FDConv 的下采样
```

同样在 `__init__.py` 和 `tasks.py` 中注册 `ConvFD`。

### C.2 Detect Head 中的卷积

Detect Head 的 `cv2`（bbox 回归分支）和 `cv3`（分类分支）通常由两个 3×3 Conv + 1×1 Conv 组成。在 YOLOv8 中，这些由 `nn.Sequential` 构建。替换方式类似——将内部的 `Conv` 替换为 `ConvFD`。

**但强烈建议**：Head 部分的 FDConv 替换放到最后验证，因为它是直接产生输出的层，FDConv 的动态特性可能破坏训练初期稳定性。建议使用 `ConvFD` 包装类并在 Head 配置中通过参数区分。
